TEAM MEMBER 1 — Project Setup, Routing & Authentication
--------------------------------------------------------------------------------

Role: Project Architect & Authentication Lead

Files Worked On:
  - src/main.jsx                    (Application entry point)
  - src/App.jsx                     (Root component, routing setup)
  - src/context/AuthContext.jsx      (Authentication state management)
  - src/services/authService.js      (Login credential verification)
  - src/services/mockData.js         (Hardcoded users & mock data)
  - src/routes/ProtectedRoute.jsx    (Route guard for logged-in users)
  - src/routes/AdminRoute.jsx        (Route guard for admin-only access)
  - vite.config.js                   (Build tool configuration)
  - index.html                       (HTML template & meta tags)

Contribution Summary:
  - Set up the React + Vite project with Tailwind CSS configuration
  - Designed the application architecture and provider nesting order
    (BrowserRouter → AuthProvider → QueueProvider → Routes)
  - Built the AuthContext using React Context API with localStorage
    persistence so login survives page refresh
  - Created the authService that matches credentials against hardcoded
    users and returns safe user objects (password stripped)
  - Implemented role-based route protection:
      * ProtectedRoute — blocks unauthenticated users, redirects to "/"
      * AdminRoute — blocks non-admin users, redirects to "/user"
  - Defined all mock data: user credentials, initial queue entries,
    and the AVG_SERVICE_TIME constant
  - Configured React Router with three routes: "/", "/user", "/admin"

Key Technical Decisions:
  - Used lazy state initialization in useState() to read localStorage
    only once on mount instead of every render
  - Used useEffect to sync auth state bidirectionally with localStorage
  - Used <Navigate replace /> to prevent guarded URLs from appearing
    in browser history

TEAM MEMBER 2 — Queue Logic & State Management
--------------------------------------------------------------------------------

Role: Core Logic & Backend Simulation Lead

Files Worked On:
  - src/context/QueueContext.jsx      (Queue state management — the core engine)
  - src/utils/queueHelpers.js         (ETA calculation & status derivation)
  - src/hooks/useQueuePosition.js     (Custom hook for queue position data)

Contribution Summary:
  - Built the QueueContext — the central engine of the entire application
    using React Context API with two state variables:
      * queue[] — array of people in the queue
      * currentServingIndex — integer pointer to who's being served
  - Implemented the three queue actions:
      * nextCustomer() — increments currentServingIndex by 1 (with guard)
      * skipCustomer() — increments currentServingIndex by 1 (with guard)
      * joinQueue(name, email) — appends a new entry to the queue array
  - Ensured the queue is strictly admin-controlled (event-driven):
      * NO timers, NO intervals, NO automatic advancement
      * Queue moves ONLY when admin clicks Next or Skip
  - Created the calculateETA() utility function:
      * Formula: ETA = peopleAhead × AVG_SERVICE_TIME (5 minutes)
  - Created the getStatus() utility function that derives status labels:
      * "Completed"    — customer index < currentServingIndex
      * "Now Serving"  — customer index === currentServingIndex
      * "Almost There" — customer index === currentServingIndex + 1
      * "Waiting"      — all others
  - Built the useQueuePosition() custom hook that combines queue context
    data with helper functions for easy consumption by components

Key Technical Decisions:
  - Queue state uses a pointer-based approach (index moves, array stays)
    instead of mutating/removing array elements — simpler and less error-prone
  - All stats (served, remaining, eta, status) are derived values computed
    on every render, NOT stored in separate state — avoids sync bugs
  - Used functional state updates: setCurrentServingIndex((prev) => prev + 1)
    to avoid stale closure issues

      * Auto-redirect if already logged in

      * Error message display for invalid credentials

      * Shows user avatar (first letter), name, and role
